CCNA Day 23 Lab: Multiple Area OSPF Configuration and Verification

Lab Objective:
The objective of this lab is to configure Multiple Area OSPF across a four‑router topology, where one router serves as the Area 0 backbone through which all other areas communicate. You will divide the network into multiple OSPF areas to improve scalability, reduce LSDB size, and optimize routing performance. This lab will strengthen your understanding of Area Border Routers (ABRs), inter‑area route exchange, and the critical role of the backbone area in multi‑area OSPF design.


Step by Step Instructions
Step 1: Build the Topology in Packet Tracer
Devices Required:
4 Routers (R1, R2, R3, R4) Cisco 2811 (Note: Add WIC‑2T modules for serial interfaces before powering on)
12 PCs (PC1–PC12)
Switches (1 per LAN segment)
Serial & Ethernet cables
Connections:
LAN 1: PC1–PC3 → Switch1 → R2 (FastEthernet0/0)
LAN 2: PC4–PC6 → Switch2 → R3 (FastEthernet0/0)
LAN 3: PC7–PC9 → Switch3 → R4 (FastEthernet0/0)
LAN 4: PC10–PC12 → Switch4 → R1 (FastEthernet0/0)

WAN Links:
R1 (Serial0/0/0) ⟶ R2 (Serial0/0/0)
R1 (Serial0/0/1) ⟶ R3 (Serial0/0/0)
R1 (Serial0/1/0) ⟶ R4 (Serial0/0/0)


Note: Power off routers before adding WIC‑2T modules for serial connections. Power them back on after modules are installed.


Step 2: Configure Router Hostnames
Router> enable
Router# configure terminal
Router(config)# hostname R-1
On R2:
Router> enable
Router# configure terminal
Router(config)# hostname R-2
On R3:
Router> enable
Router# configure terminal
Router(config)# hostname R-3
On R4:
Router> enable
Router# configure terminal
Router(config)# hostname R-4


Step 3: Configure IP Addresses on Router Interfaces
R1 (Backbone Router):
R1(config)# interface Serial0/0/0
R1(config-if)# ip address 10.0.0.1 255.255.255.252
R1(config-if)# no shutdown
R1(config)# interface Serial0/0/1
R1(config-if)# ip address 11.0.0.1 255.0.0.0
R1(config-if)# no shutdown
R1(config)# interface Serial0/1/0
R1(config-if)# ip address 12.0.0.1 255.0.0.0
R1(config-if)# no shutdown

R2 (Area 1):
R2(config)# interface FastEthernet0/0
R2(config-if)# ip address 192.168.1.1 255.255.255.0
R2(config-if)# no shutdown
R2(config)# interface Serial0/0/0
R2(config-if)# ip address 10.0.0.2 255.0.0.0
R2(config-if)# no shutdown

R3 (Area 2):
R3(config)# interface FastEthernet0/0
R3(config-if)# ip address 192.168.2.1 255.255.255.0
R3(config-if)# no shutdown
R3(config)# interface Serial0/0/0
R3(config-if)# ip address 11.0.0.2 255.0.0.0
R3(config-if)# no shutdown

R4 (Area 3):
R4(config)# interface FastEthernet0/0
R4(config-if)# ip address 192.168.3.1 255.255.255.0
R4(config-if)# no shutdown
R4(config)# interface Serial0/0/0
R4(config-if)# ip address 12.0.0.2 255.0.0.0
R4(config-if)# no shutdown


Step 4: Configure IP Addresses on PCs
LAN 1 (PC1–PC3): IP Range: 192.168.1.10 – 192.168.1.12 Subnet Mask: 255.255.255.0 Default Gateway: 192.168.1.1
LAN 2 (PC4–PC6): IP Range: 192.168.2.10 – 192.168.2.12 Subnet Mask: 255.255.255.0 Default Gateway: 192.168.2.1
LAN 3 (PC7–PC9): IP Range: 192.168.3.10 – 192.168.3.12 Subnet Mask: 255.255.255.0 Default Gateway: 192.168.3.1
LAN 4 (PC10–PC12): IP Range: 192.168.10.10 – 192.168.10.12 Subnet Mask: 255.255.255.0 Default Gateway: 192.168.10.1


Step 5: Enable OSPF Routing Protocol (Multiple Areas)
R1 (Backbone Area 0):
R1(config)# router ospf 1
R1(config-router)# network 192.168.10.0 0.0.0.255 area 0
R1(config-router)# network 10.0.0.0 0.0.0.0 area 0
R1(config-router)# network 11.0.0.0 0.0.0.0 area 0
R1(config-router)# network 12.0.0.0 0.0.0.0 area 0

R2 (Area 1):
R2(config)# router ospf 1
R2(config-router)# network 192.168.1.0 0.0.0.255 area 1
R2(config-router)# network 10.0.0.0 0.0.0.1 area 0

R3 (Area 2):
R3(config)# router ospf 1
R3(config-router)# network 192.168.2.0 0.0.0.255 area 2
R3(config-router)# network 11.0.0.2 0.0.0.1 area 0

R4 (Area 3):
R4(config)# router ospf 1
R4(config-router)# network 192.168.3.0 0.0.0.255 area 3
R4(config-router)# network 12.0.0.0 0.0.0.1 area 0


Step 6: Verify Routing Tables On all routers:
R1# show ip route
R2# show ip route
R3# show ip route
R4# show ip route
You should see OSPF‑learned routes marked with an O (intra‑area) or O IA (inter‑area).




Step 7: Test Connectivity
From PC1 → ping PC5 (192.168.2.x)
From PC8 → ping PC11 (192.168.10.x)
From PC12 → ping PC3 (192.168.1.x)
Expected Result: All pings succeed if OSPF is correctly configured and inter‑area routing is functioning.



Final Notes This lab simulates a scalable enterprise network using Multiple Area OSPF. It demonstrates how the backbone area (Area 0) acts as the central hub for inter‑area communication, reducing routing overhead and improving network efficiency.
